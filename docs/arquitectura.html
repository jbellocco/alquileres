01 — Producto

Propuesta de valor

Qué problema resuelve y para quién.

El problema

Los inquilinos no tienen acceso a información real sobre propiedades antes de alquilar. Los avisos inmobiliarios no revelan problemas frecuentes, el historial del propietario ni la experiencia real de vivir ahí.

La solución

Una plataforma donde los ex-inquilinos dejan reseñas estructuradas de las propiedades que alquilaron. Con el tiempo, cada dirección acumula una ficha de reputación basada en experiencias reales.

👤 Usuario primario

Persona que está evaluando alquilar una propiedad y quiere saber qué experiencias tuvieron otros inquilinos.

✍️ Usuario contribuidor

Ex-inquilino que vivió en una propiedad y quiere documentar su experiencia para ayudar a otros.

📊 Modelo de valor

Cuantas más reseñas tiene una propiedad, más útil es la ficha. El valor crece con el volumen de datos.

02 — Stack

Stack tecnológico

Todos los servicios son serverless y se pagan por uso, sin infraestructura fija.

Frontend
HTML + CSS + JavaScript vanilla Sin framework
Archivos estáticos servidos desde Amplify Hosting. Sin build step en el frontend.
Hosting
AWS Amplify Hosting CD automático
Deploy automático en cada push a main vía GitHub Actions + Amplify build.
API
AWS AppSync GraphQL
API GraphQL managed. Generada automáticamente desde el schema de Amplify Gen 2.
Base de datos
Amazon DynamoDB NoSQL
Tablas por modelo. Pricing on-demand: se paga por lectura/escritura, no por tiempo.
Auth
Amazon Cognito Gratis <10k usuarios
User pools para login/registro. Identity pools para permisos granulares en S3.
Storage
Amazon S3 Fotos
Bucket con paths por usuario. Fotos de reseñas y fotos de perfil. Acceso por URL firmada.
Infraestructura
AWS CDK (via Amplify Gen 2) IaC
Todo el backend definido como código en amplify/data/resource.ts. Deploy automático vía CI/CD.
CI/CD
GitHub Actions + Amplify Build
Push a main → Amplify build → backend deploy (CDK) → frontend deploy automático.

Modelo de autenticación y permisos

🔓 Usuario no registrado

Puede ver propiedades y reseñas. Opera con credenciales IAM anónimas (Cognito Identity Pool unauthenticated). Solo lectura.

🔐 Usuario registrado

Puede crear reseñas, subir fotos, y editar/borrar sus propias reseñas. Autenticado con Cognito User Pool. Lectura + escritura propia.

05 — Flujos

Flujos principales

Publicar una reseña

01

Usuario completa el formulario

Dirección, puntaje, trato, problemas, recomendación, fotos, datos del alquiler (moneda, contrato)

02

Normalización y fingerprint

Se genera un fingerprint de la dirección para detectar si la propiedad ya existe

03

Get or Create Property

Si la propiedad no existe, se crea. Si ya existe, se reutiliza el mismo registro

04

Upload fotos a S3

Hasta 5 imágenes subidas al bucket con path por usuario

05

Crear Review

Se guarda la reseña con todos los campos estructurados

06

Actualizar stats Property

Se actualizan los contadores acumulados en tiempo real (promedio, recomendación, problemas, trato)

Ver la ficha de una propiedad

01

Carga Property + Reviews

Fetch paralelo de la propiedad y todas sus reseñas activas

02

Build ViewModel

Se construye un objeto de datos preparado para UI separando fetch de presentación

03

Render condicional

Cada bloque evalúa si tiene datos suficientes (MIN_RESPALDO=2) antes de mostrarse

04

Carga async de imágenes

URLs firmadas de S3 se cargan después del render inicial para no bloquear la vista

Borrar una reseña

01

Usuario solicita borrado

Desde "Mis reseñas", el usuario puede borrar una reseña propia

02

Eliminar Review

Se elimina el registro de DynamoDB

03

Recalcular stats Property

Los contadores se decrementan. Si la propiedad queda sin reseñas, los stats vuelven a cero

06 — Lógica de negocio

Reglas del sistema

Deduplicación de propiedades (Fingerprint)

El sistema evita duplicados de la misma dirección usando una clave compuesta:

fingerprint = provincia + ":" + calleNormalizada + ":" + alturaExacta + ":" + tipoVivienda + ":" + piso + ":" + depto

La calle se normaliza eliminando acentos, convirtiendo a minúsculas y estandarizando abreviaciones ("av." → "avenida"). Esto garantiza que "Av. Corrientes 1234" y "Avenida Corrientes 1234" sean la misma propiedad.

Acumulación progresiva de datos

Los datos de la ficha de propiedad se construyen a lo largo del tiempo a partir de múltiples reseñas. Los contadores se actualizan con cada nueva reseña y se decrementan al borrar.

MIN_RESPALDO = 2 — Un dato agregado solo se muestra públicamente si está respaldado por al menos 2 reseñas. Esto evita que una sola opinión sesgada domine la ficha.
Contadores en tiempo real — No se recalcula el promedio leyendo todas las reseñas cada vez. Los stats (promedio, conteos, JSON de problemas) se actualizan incrementalmente en el registro de Property al crear o borrar una reseña.
Altura pública vs exacta — Para proteger la privacidad, se guarda la altura exacta internamente pero se muestra solo un rango redondeado al público (ej: "6100" → se muestra como referencia de cuadra).

Georeferenciación

El formulario de nueva reseña usa la API Georef Argentina (georef.api.gob.ar) para autocompletar y validar provincias, partidos y barrios. Esto garantiza datos geográficos consistentes y correctamente escritos.

07 — Pantallas

Páginas de la aplicación

🏠 index.html — Inicio

Listado de propiedades con búsqueda por texto y filtros. Cards con rating, cantidad de reseñas y señales principales.

🏢 propiedad.html — Ficha de propiedad

Vista principal centrada en Property. Hero con imagen y métricas. Sidebar con KPIs. Datos útiles agregados. Listado de reseñas como respaldo.

✍️ nueva-resena.html — Nueva reseña

Formulario completo. Dirección con autocompletado Georef. Calificación, trato, problemas, fotos. Datos opcionales: moneda, precio, tipo de contrato.

📋 mis-resenas.html — Mis reseñas

Panel del usuario logueado. Lista de reseñas propias con opción de borrar. Actualiza los stats de la propiedad al borrar.

🔐 login.html / registro.html

Autenticación via Cognito. Login con redirect param (si el usuario llega desde una página protegida, vuelve a ella después de loguear).

👤 perfil.html — Perfil de usuario

Edición de nombre, foto de perfil y datos de contexto del usuario.
